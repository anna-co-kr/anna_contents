## 0. 임원 요약 (한 문단)

Prompt Studio v0.5는 **도구 자체가 목적이 아니라**, 안나의 글로벌 디자인 큐레이터 채널의 첫 영상 (Birdie 커버) 제작을 2주 안에 성사시키기 위한 **창작 속도 레버리지 인프라**다. 어제 V1 A (Reel Studio v1) 설계의 핵심 결론 — "영상 1편이 1순위 산출물, 도구는 그 1편을 돕는 선까지" — 는 유지한다. 바뀐 것은 세 축이다: (1) 채널 IP 포맷이 **Y (목소리·얼굴 모두 X)** 에서 **D형 (마스코트 + 안나 목소리 내레이션)** 으로 피봇, 근거는 AI 에셋 포화 시대의 휴먼 터치 메커니즘 (김햄찌 사례). (2) 도구 스코프가 "Reference Library v0" (URL 수집·태그·임베딩)에서 **Prompt Studio v0.5**로 확장, 핵심 엔진이 프롬프트 수렴으로 이동, 이미지 ↔ 프롬프트 페어 로그와 토큰 diff 트래커 추가. (3) V1 첫 영상 타겟 층이 "EU/NA 디자인 큐레이터"에서 **한국 인바운드 관광객**으로 이동, Havea §3.7 유통 레이어와 자연 synergy 근거. 강의 기한 제약으로 **도구는 반드시 만든다**, 단 스코프는 영상 제작 레버리지 범위로 한정.

## 1. 문제 정의

### 1.1 진짜 문제 (어제 V1 A에서 이월, 살짝 확장)

1. **상류 (아직 검증 안 됨)**: 안나의 감도·큐레이션·브랜드 서사 능력이 **글로벌 영어권 오디언스**에게 닿는가? 특히 K-친화 하이엔드 취향층.
2. **하류 (도구가 다룰 영역)**: 솔로·영상 경험 거의 없음·편집 제약 상태에서 어떻게 **지속 가능한 제작 리듬**을 만드는가? — 오늘 세션에서 구체화: **AI 생성 에셋 (이미지·영상)의 프롬프트 수렴 병목**이 1편당 2시간짜리 반복 작업으로 측정됨.

### 1.2 어제 doc과 오늘 doc의 관계

어제 doc (V1 A)은 "영상 1편 먼저, 도구는 반응 본 뒤"로 프레이밍했다. 오늘 doc (V1 B)은 안나의 현실 제약을 반영해 **"영상 1편 + 도구 1모듈 동시 진행"**으로 조정한다. 단 도구는 영상 제작에 **직접 박혀 사용**되는 것만 포함. 도구가 자기목적적이 되는 순간 프로젝트 실패.

## 3. 현재 상태 (Status Quo)

안나가 V1 B 전에 실제로 돌리고 있는 워크플로우:

- **툴 스택**: 캡컷 (기초), Midjourney, Nano Banana, Higgsfield, Photoshop Web 무료. AE·Premiere·정식 Photoshop 없음.
- **제작 경험**: 이미지 콘텐츠 위주. 영상 거의 처음.
- **최근 실측 (버디 코리아 포스트, 이미지 캐러셀)**:
  - AI 타이틀 이미지 반복 생성: 2시간
  - 손보정·레이아웃: 15-30분 (추정)
  - 캡션 작성: 30분-1시간 (추정)
  - 업로드·해시태그: 5-10분
  - **합계**: 3-4시간
- **최근 실측 (버디 릴스)**:
  - 기존 @birdie.official 본사 계정 리그램
  - AI 타이틀 이미지 생성: 2시간
  - 한글 자막 추가 + 편집: 1-2시간
- **프롬프트 병목 원인 (안나 본인 답변)**: "어떤 단어·명사·스타일 키워드를 넣어야 할지 실험" = 네 가지 중 **프롬프트 자체**가 2시간의 주범 (스타일 일관성·타이틀 문구·선별 아님)
- **대화형 AI 작업 선호** — 폼 기반 UX 피로감, Claude 대화형 기획 경험 좋아함

### 3.1 오늘 발견된 핵심 패턴 — "평가 속도 ≫ 표현 속도"

안나는 취향·감도의 축이 명확하다 (Indieshop·Scottie Cameron·Jacquemus·Simpsons 콜라주 결, 어제 doc §15에서도 관찰됨). "이건 맞고 이건 틀리다" 판단은 빠름. 그런데 "이걸 어떤 단어로 묘사해야 Midjourney가 뽑는지"는 느림. **평가 속도가 표현 속도를 앞지르는 상태**. 이게 2h 프롬프트 실험의 구조적 원인이고, 이게 도구가 메우는 gap이다.

## 4. 타겟 유저 & 가장 좁은 웨지

### 4.1 채널 포지셔닝 (오늘 브리프 §3.1 수용, 어제 §4.1 수정)

- **어제 V1 A**: "Seoul-based global indie curator (서울에서 보는 글로벌 디자인)"
- **오늘 V1 B**: **"하이엔드 취향의 K-친화 글로벌 영어권 큐레이션 채널"** (브리프 §3.1 그대로)
- **변경 이유**: 어제 포지셔닝은 EU/NA 디자인 큐레이터층만 상정해 Havea §3.7 유통 레이어와 연결성 약했음. 오늘 포지셔닝은 "K-친화" 축을 명시해 인바운드 관광객부터 K-wave 팬덤까지 한 층 아래 담을 수 있음.

### 4.2 V1 B 첫 영상 타겟 (P9'' 합의: 한국 인바운드 관광객)

**왜 인바운드 관광객인가** (안나 논거):
1. 이미 **"서울 쇼핑·여행" 검색·해시태그 행동**이 존재하는 집단 = 자연 유입 경로
2. Havea §3.7 "서울 동네 큐레이션" 유통 레이어와 직결 (커머스 전환 잠재력 최고, 브리프 §3.4 후반 발견 레이어)
3. Birdie "덴마크 브랜드를 서울에서 산다" 각도와 메시지 자연스러움
4. 대부분 아시안계, 서양인도 "아시아 관심층" = 안나의 K-친화 톤과 fit

**단서** (design doc 기록, 실행 시 유의):
- "관광객"은 타겟이자 **유입 경로 전략**. 콘텐츠 앵글은 "브랜드 숨은 이야기 (§3.5)"와 "서울 방문시 가이드"를 **접합**. 예: "서울에서 발견한 이 작은 덴마크 기기가 왜 이 아파트 안 공기를 다시 읽게 만들었는가" > 단순 "Birdie 브랜드 소개"
- 다음 확장 경로 (동남아/남미 순차)는 **V1 48h/7d/14d 반응 데이터 보고 Phase 2에서 결정**. 지금 확정 안 함.

### 4.3 cross-model challenge & 안나 방어 (Phase 3.5 기록)

Claude subagent가 P9''에 대해 "인바운드 관광객 = 1회성 트래픽, 재방문/구독 루프 약함. A/B 완료율 테스트로 증명 필요, 아니면 EU/NA 또는 한국어 디자인 네이티브 커뮤니티로 회귀" 라고 challenge.

**안나 방어**: 유입 경로 자연스러움 + Havea 유통 synergy를 근거로 P9'' 유지. 단 A/B 실험 (Week 1 D13 한국어 원본 vs 영어 자막 인바운드 버전 동시 발행)은 V1 스코프에 추가하지 않기로 결정. 리스크는 수용.

**리스크 flag** (doc에 명시): A/B 실험 없으면 P9''가 틀렸을 때 1편 데이터만으로 판단해야 함. 반응 애매할 경우 (저장 높은데 완료율 낮음 등) 원인 분리 어려움. V1.5에서 2편째 다른 각도 시도로 데이터 보강 필요.

### 4.4 가장 좁은 웨지 (Narrowest Wedge)

**@havea.official 계정에 Birdie 커버 영상 1편 (60-90초).** 어제 V1 A 그대로 유지.

- **왜 가장 좁은가**: 1 계정 × 1 영상 × 1 브랜드
- **왜 Birdie인가**: (a) 안나가 수입 권리 보유 = 100% 통제, (b) 덴마크 origin = 인바운드 관광객 타겟 "서울에서 산 덴마크 브랜드" 각도와 fit, (c) 기존 @havea.official 팔로워에게 친숙 앵커
- **왜 @havea.official**: 기존 10K 활성 팔로워 = 0에서 시작 아님. 기존 팔로워 이탈 리스크는 V1이 측정

## 5. 제약 조건

### 5.1 자원 제약

- **인력**: 솔로 (안나 1인)
- **시간**: 3주 (강의 기한 + 영상 출시 + 도구 완성 동시)
- **예산**: 기존 구독 외 신규 비용 최소. Anthropic API 학생 크레딧.
- **편집 툴**: 캡컷만 (AE·Premiere 사용 불가)

### 5.2 기술 제약

- **캐릭터 일관성**: MJ `--cref`, Nano Banana character mode 모두 의상·조명·디테일 컷마다 흔들림. LoRA 학습은 시간·비용 추가.
- **한국어 네이티브**: 안나는 영어 네이티브 아님. 영어 오디오 제작 시 전달력 저하 리스크 (→ 자막 중심 해결)

### 5.3 포맷·운영 제약

- **최종 영상 포맷 = D형** (마스코트 + 안나 목소리 내레이션). 얼굴 노출 X, 목소리 노출 O. 근거: 김햄찌 메커니즘 (AI 비주얼 + 인간 목소리 + 리얼리티 스토리텔링)
- **계정 = @havea.official** (기존 30K, 한국 중심). 신규 채널 아님
- **오디오 전략**: **한국어 내레이션 + 영어 자막 메인 + 한국어 자막 서브**. 인바운드 관광객 타겟에 영어 자막 필수
- **BGM·사운드 디자인**: Week 1 D10 단계에서 결정 (Artlist·Epidemic Sound 트라이얼 후보)

### 5.4 강의·학습 제약

- 짐코딩 "비개발자를 위한 클로드 코드" 강의 과제 마감 존재 (세부 일정은 안나 확인 필요)
- 스택 학습 곡선: Next.js 15 + Supabase + shadcn + Tailwind + Vercel + pgvector. 안나 현재 학습 중
- **gstack 스킬 파이프라인** 학습도 함께 진행: `/office-hours → /autoplan → 구현 → /review → /qa → /ship`

### 5.5 철학 제약

- **스토리텔링·큐레이션·편집의 최종 감도 판단은 안나**. AI는 1차 후보·초안까지만 (P6' 참조)
- **대화성 보존** — 안나가 재미있어하는 AI 대화형 기획 경험 유지. 폼 UX 최소화
- **IP는 결과물이 아닌 전제 조건** — V1 A에서 승격된 전제 유지. D형 마스코트가 없으면 첫 영상 못 찍음

### 9.3 Prompt Studio v0.5 상세 스코프

**목적**: 프롬프트 수렴 병목 해결 → 버디 영상 제작 레버리지. 안나 실 워크플로우 시간·이터레이션 축소.

**모듈 경계 (중요, P2 "1모듈" 약속과 정합)**:
- **V1 코어 (기능 1+2+3)** — D13 출시 전 반드시 동작. 드롭·태그·페어 로그. 이것만 되면 강의 제출 가능
- **V1.5 (기능 4+5+6)** — Week 3 (D15-D21) 또는 V1 반응 본 뒤 추가. 토큰 diff·벡터 검색·리믹스. 중요하지만 non-blocking
- **판단 기준**: D10에 V1 코어 3개 기능 미완이면 나머지는 전량 V1.5로 밀어 영상 출시(D13) 지킴. **크리티컬 패스 = 영상 1편, 도구 6기능 아님.**

**Definitions 블록** (이하 측정·기록에 일관 사용):
- *만족 결과* = 안나가 "이 이미지를 최종 포스트 후보군에 넣겠다"로 체크한 생성 이미지
- *이터레이션 1회* = MJ `/imagine` 또는 variation 호출 1건 (프롬프트 문구 변경 여부와 무관하게 1회로 카운트)
- *수렴* = 만족 결과 첫 발생까지의 이터레이션 누적 수

**기능 상세**:

1. **URL/이미지 드롭 → 메타·썸네일·구조화 토큰 분해** (Vision 자동)
   - IG/Pinterest/YouTube/Are.na URL 입력 → `/browse` 스킬로 썸네일·메타 수집
   - 이미지 업로드도 지원 (drag-drop)
   - Claude Vision이 각 이미지를 6차원 구조화 토큰으로 분해: `subject` / `style` / `lighting` / `composition` / `medium` / `mood`
   - 예: `{subject: "Danish CO2 sensor on wooden table", style: "editorial Nordic minimalism", lighting: "soft northern window light", composition: "off-center negative space", medium: "35mm film look", mood: "quiet attention"}`

2. **좋아요·태그·프롬프트 스니펫 필드**
   - 각 레퍼런스에 ❤️ 점수 (1-10), 수동 태그 (카테고리·무드·색감·용도)
   - **프롬프트 스니펫 필드**: 이 레퍼런스 느낌 뽑을 때 사용한 프롬프트 저장. 여러 버전 가능
   - [copy prompt] 버튼 → 클립보드 → MJ에 바로 붙여넣기

3. **Prompt ↔ Result pair 로그** (신규, subagent 제안 수용)
   - 안나가 MJ에서 실제 돌린 프롬프트 + 결과 이미지를 pair로 저장
   - "만족 결과" 마킹 기능 (별점 또는 체크)
   - 한 프롬프트에 여러 결과 이미지 연결 가능 (MJ variations)

4. **토큰 diff UI** (신규, subagent 제안 수용)
   - 성공한 프롬프트 vs 실패 프롬프트의 **토큰 차이** 시각화
   - 예: "이 프롬프트에서 'muted' 추가하니 결과 확 달라짐" 같은 추적
   - MVP는 심플: 시간 순 나열 + 차이 단어 하이라이트

5. **pgvector 유사 레퍼런스 검색**
   - 구조화 토큰 기반 임베딩
   - 새 레퍼런스 드롭 시 "유사 테이스트 성공 프롬프트 N개" 자동 추천
   - 검색 UI (태그 필터 + 벡터 검색 조합)

6. **리믹스 제안** (Claude 대화형)
   - 저장된 레퍼런스 선택 → "이 느낌 × 새 주제" 프롬프트 변형 후보 3개 Claude가 생성
   - 복붙 → MJ → 결과 저장 (3번 기능으로 연결)

**비기능 (명시)**:
- **이미지·영상 생성 없음** — 생성은 여전히 MJ·Nano Banana. 도구는 축적·재사용·추천·역공학만
- **MJ API 호출 없음** — MJ는 수동 복붙 워크플로우 유지 (API 비용·승인 복잡도 회피)
- **추천 엔진 없음** — taste graph recommendation은 Phase 2 (데이터 축적 필요)
- **자동 자막 생성 없음** — 캡컷 기본 기능 사용
- **성과 측정 대시보드 없음** — 플랫폼 네이티브 인사이트 사용

### 9.3.1 Tech Appendix (가정치, `/plan-eng-review`에서 재확정)

필수 엔지니어링 결정의 초안. 예산·스키마 충돌 사전 감지 목적. 정식 설계는 eng review에서.

**고정 토큰 스키마** (JSONB 컬럼):
```json
{ "subject": "", "style": "", "lighting": "", "composition": "", "medium": "", "mood": "" }
```
6-key shape 엄격 고정. Vision 응답을 structured output (tool_use / JSON mode)으로 강제해 스키마 드리프트 방지.

**Supabase 테이블 개략 (ERD 1장 분량)**:
- `references` (id, source_url, thumbnail_url, uploaded_at, user_id, favorite_score, notes)
- `reference_tokens` (reference_id FK, tokens jsonb, embedding vector(1024), vision_model_version, created_at)
- `tags` (id, reference_id FK, tag, tag_kind [category/mood/color/purpose]) — PRD가 source of truth. `purpose`는 "용도" 카테고리 (예: "제품 샷", "공간 무드", "인물 배경")
- `prompts` (id, reference_id FK NULL, prompt_text, source [copied/modified/remix], created_at)
- `pairs` (id, prompt_id FK, result_image_url, satisfaction [null/1-5], is_final_pick bool, iteration_count_cumulative, session_id, created_at)

**임베딩 선택**: Claude Vision은 임베딩 API 미제공. Voyage-3 (1024 dim) 또는 OpenAI text-embedding-3-small (1536 dim) 중 택1. 비용·한국어 토큰 처리 고려해 **Voyage-3 기본 추정**. eng review에서 확정.

**Vision 호출 비용 캡**: Claude Sonnet Vision 이미지당 약 $0.005-0.015 (해상도 의존). 일 사용 상한 $3 = 약 200-600 이미지/일. 월 $50 학생 크레딧 내 여유. **안전장치**: 일 한도 초과시 token 분해 기능 비활성 (수동 태그로 폴백).

**Rate limit·에러**: 단일 사용자라 rate limit 충돌 낮음. Vision 호출 실패시 재시도 1회 → 실패 persist → 수동 태그 UI 노출.
