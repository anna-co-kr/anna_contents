## 4. 구체적 페인 (안나 원문 인용)

> ⚠️ **office-hours가 반드시 읽어야 할 섹션.** 가설이 아니라 안나 본인의 자기 관찰임.

### Pain 1. 감도 있는 영상이 만들기 어려움 (root pain)

> *"감도있는 영상을 만드는것도 나의 페인포인트이거든..!"*

### Pain 2. 머릿속 영상을 언어로 번역 못함

> *"머릿속에 있는 영상에 대한 내용을 말로 설명하는게 어려운거같아."*

이게 지속적으로 Pain 3을 유발하는 상류 페인임.

### Pain 3. 프롬프트 실패 반복

> *"프롬프트를 작성해서 이미지를 만들었는데, 원하는 이미지가 나오지 않아서 계속 시간을 소비하는 게 제일 지겨운 거같아!"*

### Pain 4. 모션 그래픽 언어 자체를 모름

> *"영상에서 고감도를 끌어내려면 에프터에펙트같은 툴 사용이 제일 막막하고"*
> *"모션 작업은 또 한번도 해본 적이 없어서 어떻게 작업해야할지도 모르겠거든. 그리고 모션작업이나 영상 편집하는 과정자체를 내가 잘 모르기때문에 프롬프트를 어떻게 활용해서 써야하는지도 모르겠어"*

### Pain 5. 스토리텔링 기획의 무게

> *"우선 영상을 만들때 가장 공들이는건 스토리텔링이 아닐까 싶어. 마치 소설책을 읽는 것 같은 몰입감을 선사하면서도 절대적으로 계속 보고 싶게 만드는 스토리가 제일 중요하니까!"*

스토리텔링은 **안나가 공들이는 강점 영역**. 자동화 대상이 **아님**. 오히려 이걸 뒷받침해주는 다른 단계의 효율이 필요함.

### Pain 6. 1초 후킹 스토리텔링 약함

> *"타겟에게 1초안에 후킹 될만한 스토리텔링"*

### Pain 7. 재미있는 부분도 있음 (동기 보존)

> *"영상에 대해서 AI 랑 대화하면서 아이디에이션하고 기획하고 하는 이런 내용은 정말 재미있는거같아!"*

이건 **살려야 할 부분**. 도구가 대화성을 잃으면 안나가 지루해 함.

## 6. 영상 제작 22단계 표준 파이프라인

안나의 문제를 풀기 위한 전체 지형. 이 파이프라인이 본 도구의 **구조적 뼈대**.

### Stage 1 — 기획 (Pre-Production)
1. 아이디어·컨셉
2. **리서치·레퍼런스** ← 글로벌 감도 채우는 지점 (Pain 1, 2)

4. **스크립트·내레이션**
5. **샷 리스트·스토리보드**
6. 스타일 가이드

### Stage 2 — 자산 준비 (Assets)
7. **이미지 생성** ← Midjourney, Nano Banana (Pain 3)
8. **영상 생성·촬영** ← Higgsfield, 직접 촬영
9. B-roll 수집
10. 보이스·내레이션
11. BGM 선곡

### Stage 3 — 편집 (Post-Production)
12. 러프 컷
13. 파인 컷
14. **모션 그래픽·타이포** ← AE 대체 (Pain 4, Phase 3)
15. 색보정
16. 자막·캡션
17. 사운드 믹싱

### Stage 4 — 배포 (Distribution)
18. 썸네일·커버 이미지
19. 제목·설명·해시태그
20. 플랫폼별 변형

22. 성과 측정·피드백

## 7. MVP 스코프 (2주)

### 7.1 포함 (Must-Have)

**Stage 1의 3~5번 + Stage 2의 7번** = "기획-자산 브리지"

구체적으로:

- **M1. 컨셉 → 훅 생성기**: 주제·브랜드·제품 입력 → 1초 후킹 훅 카피 10개 자동 생성 → 좋아요·싫어요 투표
- **M2. 훅 → 스크립트·스토리보드**: 선정된 훅을 기반으로 스크립트·샷 리스트·컷별 비주얼 설명 자동 생성 (대화형, 되묻기 포함)
- **M3. 스토리보드 → 이미지 프롬프트 공장**: 각 컷의 비주얼 설명 → Midjourney·Nano Banana·Higgsfield 각각에 최적화된 프롬프트 자동 조립 → 한 번에 10~30개 시안 생성 → 좋아요·싫어요 투표 → 재생성
- **M4. Taste Graph 인프라**: 위 모든 단계의 좋아요·싫어요·저장·삭제가 시그널로 축적 (pgvector 임베딩) → 쓸수록 M1~M3 결과가 "안나스럽게" 수렴
- **M5. 레퍼런스 프로비저너 (웹서핑 기반)**: Claude Code의 `/browse` 스킬 활용. YouTube·Instagram 공개 URL 분석, 글로벌 크리에이터 자동 모니터링, 안나 Taste Graph × 글로벌 트렌드 교집합 추천
- **M6. Pinterest 통합** (아이디어 단계, 실현 가능성 office-hours에서 확인): 안나의 Pinterest 보드·핀을 Taste Graph 초기 시드로 흡수

### 7.2 제외 (Non-Goals, 명시적으로)

- 영상 편집 자동화 (Stage 3, Phase 2)
- 모션 그래픽 생성 (Stage 3의 14번, Phase 3)
- 동시 업로드·멀티캐스팅 (Stage 4, Phase 2)
- 댓글·DM 자동 응답 (Mirr의 영역, 안 함)
- SaaS화·멀티 테넌시 (개인용)
- 다른 크리에이터 대상 판매

### 7.3 설계 원리 (철학)

- **감도 자동화 X, 판단 효율 증폭 O** — AI는 후보를 뽑고, 안나가 선택
- **IP는 결과물** — 캐릭터·스타일·톤은 Taste Graph에 축적된 결과로 드러남. 먼저 정의하지 않음
- **안나가 주인공** — 스토리텔링과 선택은 안나, AI는 번역자와 증폭기 역할
- **쓸수록 똑똑해짐** — 한 모듈의 시그널이 모든 모듈을 개선
- **대화성 보존** — 안나가 재밌어하는 대화형 기획 경험 유지

## 8. 차별화 (vs Mirr / Buffer / Later)

| 차원 | Mirr | Curator Cockpit (폐기) | **Reel Studio (이번 버전)** |
|---|---|---|---|
| 핵심 자산 | 브랜드 페르소나 1개 | N개 브랜드 DB | **Taste Graph (취향 시그널)** |
| 목표 | 콘텐츠 자동 생성 | 콘텐츠 자동 변형·배포 | **영상 제작 여정 전체 지원** |
| 사용자 관점 | 브랜드 오너 | 큐레이터 | **크리에이터 (영상 제작자)** |
| 시간 지남에 따라 | 페르소나 고정 | 브랜드 엔트리 누적 | **도구 자체가 개인화** |
| AE 대체 | 불가 | 범위 밖 | **우회 (AE 없이 가능한 미학 강화)** |
| 웹 기반 레퍼런스 | 없음 | 없음 | **있음 (/browse + Pinterest)** |

## 9. 기술 스택 (잠정)

- **프레임워크**: Next.js 15 (App Router, Server Actions)
- **언어**: TypeScript (strict)
- **DB**: Supabase (Postgres + **pgvector** + Auth + Storage)
- **UI**: shadcn/ui + Tailwind CSS
- **AI**: Anthropic Claude API (Claude Sonnet 4.6 / Haiku 4.5 라우팅)
- **이미지 생성 연동**: Midjourney, Nano Banana, Higgsfield API (각각 API 제공 여부 확인 필요)
- **Pinterest**: Pinterest API v5 (OAuth 2.0, 보드·핀 읽기, 실현 가능성 office-hours에서 검증)
- **웹서핑**: gstack `/browse` 스킬 (내장 Chromium)
- **배포**: Vercel + Vercel Cron
- **워크플로우**: Claude Code + gstack

## 10. 열린 질문 (office-hours가 찢을 지점)

> office-hours 에이전트에게: 이 섹션의 질문들을 **반드시** 포싱 질문으로 재구성해서 안나에게 던져주세요.

### Q1. 첫 모듈의 순서
M1(훅)부터 만들지, M3(이미지 프롬프트)부터 만들지, M5(레퍼런스)부터 만들지. 피드백 루프와 Taste Graph 씨앗 뿌리기 관점에서 어느 게 우선?

### Q2. Taste Graph 초기 시드
첫날 Taste Graph는 비어있음. 무엇으로 채울 것인가?
- (a) Pinterest 보드 import (안나의 기존 취향 흡수)
- (b) 글로벌 레퍼런스 100~150개 선큐레이션 (안나가 좋아요·싫어요로 프로파일링)
- (c) 감도 레퍼런스 3개(인디샵·심슨·Jacquemus) 분석으로 초기 프로파일 부트스트랩
- (d) 전부 조합

### Q3. Pinterest 통합 깊이
Pinterest API로 실제로 무엇까지 가능한가? OAuth 허들은? Rate limit? 대안(RSS, 공개 URL 스크래핑)?  
이 질문은 **기술 검증 필요** — office-hours가 `/browse`로 Pinterest Developer 문서 확인 바람.

### Q4. 웹서핑 레퍼런스 에이전트의 작동 방식
- 어느 플랫폼 모니터링? (YouTube, Instagram, TikTok, Vimeo, Behance, Are.na?)
- 주기는? (실시간·일간·주간?)
- 자동 큐레이션 기준? (조회수? 안나 Taste Graph 유사도? 글로벌 지역 필터?)
- 저작권·크롤링 에티켓 검토

### Q5. 영상 제작 순서 — 강제 vs 자유
- (a) 사용자가 Stage 1 → 2 → 3 순서를 강제로 따라가는 UX
- (b) 사용자가 원하는 단계로 점프 가능 (추천만)
